1. Introduction & Core Concepts

Instruction: Read the definitions below. These concepts help you write code that can handle unknown data inputs without crashing.

Programs must be flexible enough to repeat tasks dynamically and robust enough to handle unexpected errors gracefully.

💡 Key Definitions for Your Exam

Variables: Named memory locations where data values can be stored and changed while the program is running.

Constants: Named memory locations holding data values that cannot be changed while the program is running. (In Python, we use uppercase names to show a value is a constant).

Condition-Controlled Iteration: A loop that repeats based on a Boolean condition (like a while loop). It keeps running until the condition changes state.

Exceptions: Runtime errors (like dividing by zero) that disrupt the normal execution flow. We use exception handling to catch errors safely.

2. Python Code Examples

A. Variables and Constants

The constant stays fixed. The variable changes dynamically as the program processes calculations.

# Constant (UPPERCASE)
PI = 3.142 

# Variable (lowercase)
radius = 5
area = PI * (radius ** 2)

radius = 10 # This variable changes
B. Condition Loop (while)

Repeats as long as the condition evaluates to true. You must modify the variable inside the loop to avoid an infinite loop.

user_input = ""
while user_input != "exit":
    user_input = input("Enter word: ")
    print(f"You typed: {user_input}")
C. Exception Handling

The program tries to run code inside the try block. If a specific error happens, it runs the except block instead of crashing.

try:
    num = int(input("Enter integer: "))
    result = 10 / num
except ZeroDivisionError:
    print("Error: Cannot divide by 0")
except ValueError:
    print("Error: Must be a number")
